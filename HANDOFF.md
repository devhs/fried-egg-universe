# HANDOFF — 대화 요약 (이어서 작업할 사람을 위한 기록)

원래 요청: "계란 후라이"를 주제로 신박한 것을 만들기 → 점점 확장됨.

## 진행 흐름
1. **단발 아이디어**: 굽기 시뮬레이터 / 성격테스트 / 세계 도감 / 과학 가이드 제안.
   → 사용자가 "전부 총망라한 웹 서비스"를 원함 → 단일 HTML 데모(`fried-egg-universe.html`, 초기 버전) 제작.
2. **풀스택 전환 요청**: AWS EC2 서비스 + 리더보드 DB + 굽기 게임에 타이밍 요소 + 도감을 수집형(동일 유저 ID)으로 + 모바일 우선.
   → 확정: SQLite / 닉네임+자동ID / 전체 코드+배포가이드.
   → Node+Express+SQLite 풀스택으로 재작성. 굽기=타이밍 클릭 게임, 도감=가챠, 리더보드, 닉네임 온보딩. 통합테스트 20개 통과.
3. **오케스트라 이벤트화**: 악기 파트 선택(9종) / 닉네임 기기고정+중복금지 / 카톡 등 링크 점수공유.
   → 확정: 기기고정+중복금지, 파트표시+파트별순위, 기본공유(Web Share).
   → 백엔드(parts.js, users.part, 닉 잠금·유니크, 파트별 리더보드, 공유 카드 API) + 프론트(파트선택/태그/필터, 공유, 도전장 랜딩) 추가. 통합테스트 33개 통과.
4. **배포 자동화**: 사용자가 "SSH 정보 주면 네가 다 해줄 수 있냐" 질문.
   → 직접 SSH 불가(보안+샌드박스 제약) 설명. 대신 **deploy-al2023.sh 원샷 스크립트** + QUICKSTART-EC2.md 제작.
   → 확정: Amazon Linux 2023 / 도메인+HTTPS / GitHub clone.
5. **이 export**: 다른 Claude Code PC에서 이어가기 위해 CLAUDE.md + HANDOFF.md 작성.

## 알아둘 제약 / 메모
- 검증 샌드박스는 외부 네트워크 제한 → better-sqlite3 prebuilt 다운로드 불가, 외부 SSH 불가.
  그래서 테스트는 Node 22 내장 `node:sqlite`로 대체 검증함(운영은 better-sqlite3 그대로).
- 배포 스크립트는 bash 문법검사만 통과, 실제 EC2 실행은 미검증.
- 사용자 작업지침: 추측 금지·1차 출처·전수검사 지향. 과학 수치는 American Egg Board 출처 사용
  (흰자 62–65°C / 노른자 65–70°C 응고).

## 바로 다음 액션 (사용자 쪽)
1. GitHub 빈 저장소 생성 → 로컬에서 push
2. EC2 보안그룹 80/443 오픈, 도메인 A레코드 → EC2 IP
3. EC2에서 deploy-al2023.sh 실행 → 에러나면 출력 공유해 디버깅

## v2 세션 (다른 Claude가 이어받아 개선)
요청: 코드품질·UX / 보안·안정성 / "있어보이게 세련되게" / **로컬에서 실행** / 기능 풍부하게.

### 한 일
1. **로컬 실행 무빌드화 (핵심)** — db.js를 드라이버 자동선택으로 재작성: better-sqlite3가 빌드돼 있으면 사용,
   아니면 Node 22.5+ 내장 `node:sqlite`로 폴백(빌드도구 불필요). probe(`new Better(':memory:')`)로 실동작 확인 후 결정.
   package.json에서 better-sqlite3를 **optionalDependencies**로 이동 → 빌드 실패해도 `npm install`이 안 깨짐.
2. **보안·안정성** — 보안 헤더(CSP/X-Frame/nosniff 등), rate limit(ratelimit.js, score/draw/register/daily),
   닉네임 검증(validate.js: 제어문자 제거·길이·문자셋·욕설 블록리스트), `/api/*` JSON 404, 중앙 에러 핸들러.
3. **신규 기능** — 섹션 대항전(GET /api/parts/leaderboard, 파트별 합산/평균/최고/인원),
   일일 출석(POST /api/daily, 연속 streak 보너스, KST 기준), 도감 100% 1회성 보너스(draw 응답 collectionBonus).
4. **UI 리디자인** — 프리미엄 테마(글래스/그라데이션/희귀도 글로우/세그먼트 토글/진척바),
   컨페티(전설·신기록·출석·100%), 코인 카운트업, 탭 인디케이터.
5. **테스트** — test-api.js 33→**58개**(신규 엔드포인트·검증·rate limit 단위·404·보안헤더). node:sqlite 폴백으로 실서버 기동해 전부 통과.

### 검증 커버리지 (정직 보고)
- ✅ 확인: 58/58 테스트(node:sqlite), 클린 설치→기동, 정적 서빙, app.js 문법, app.js↔index.html id 대조.
- ⚠️ 미확인: better-sqlite3 실제 빌드/운영 경로 실행, EC2 실배포, 브라우저 픽셀단 렌더링(비주얼).

### 작업 환경 메모(중요)
- 이 환경의 bash 마운트는 **큰 파일(>~4KB)을 끝까지 동기화 못 함** → 검증은 /tmp에 복제본 만들어 수행했음.
  납품 파일(저장 폴더)은 파일 도구로 쓴 전체 버전이라 정상. 다음 사람은 큰 파일 bash 검증 시 이 점 유의.
